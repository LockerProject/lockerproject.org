A Change in Plans

There used to be some downloadable tarballs containing more-or-less self-contained builds of the locker code. The lockerballs were built for distribution at OSCon 2011, and contained support binaries that were hacked up to be semi-statically linked. In other words, they were kinda busted and not meant for long-term use.

Fortunately, if you want an easy way to get started with The Locker Project, there's a better solution. Assuming you have Python 2.6, Git, and a working C++ build environment, you can use lockerbox, a shell script that will download, build and install all the requisite components to give you a fully-functional environment for playing with—and extending—a locker of your own!

Once you've set up a locker, you'll probably want to check our wiki for documentation on how to get started with your new locker, both as a user and as a developer.
